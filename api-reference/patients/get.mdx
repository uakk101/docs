The Retrieve Patient endpoint returns the complete record for a single patient identified by their platform-assigned ID. You can call this endpoint with any API key scoped to `read`, `read-write`, or `admin` — it is safe to use in read-heavy workflows such as pre-visit chart pulls, eligibility checks, or dashboard lookups without risk of modifying data. The patient ID is the `pat_`-prefixed value returned when the record was first created.

## Endpoint

```
GET https://api.codecraftinghub.com/v1/patients/{id}
```

<Note>
  **Scope required:** `read`, `read-write`, or `admin`. Include your API key
  as `Authorization: Bearer YOUR_API_KEY` on every request.
</Note>

---

## Path Parameter

<ParamField path="id" type="string" required>
  The platform-assigned unique identifier for the patient. This value is always
  prefixed with `pat_` (e.g., `pat_01HX9Z2K3M4N5P6Q7R8S9T0U`). You receive
  this ID in the response body when you create or search for a patient.
</ParamField>

---

## Example Request

<CodeGroup>

```bash cURL
curl https://api.codecraftinghub.com/v1/patients/pat_01HX9Z2K3M4N5P6Q7R8S9T0U \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

patient_id = "pat_01HX9Z2K3M4N5P6Q7R8S9T0U"
url = f"https://api.codecraftinghub.com/v1/patients/{patient_id}"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
}

response = requests.get(url, headers=headers)

if response.status_code == 200:
    patient = response.json()
    print(f"{patient['first_name']} {patient['last_name']}")
elif response.status_code == 404:
    print("Patient not found.")
```

</CodeGroup>

A successful request returns `200 OK` with the full patient object.

```json

  "id": "pat_01HX9Z2K3M4N5P6Q7R8S9T0U",
  "mrn": "MRN-00042",
  "first_name": "Jane",
  "last_name": "Smith",
  "date_of_birth": "1985-03-14",
  "gender": "female",
  "email": "jane.smith@email.com",
  "phone": "555-867-5309",
  "address": {
    "line1": "123 Main St",
    "city": "Austin",
    "state": "TX",
    "zip": "78701"
  },
  "external_id": "EHR-PAT-9871",

  "updated_at": "2024-01-15T10:30:00Z"

```

<ResponseField name="id" type="string">
  Platform-assigned unique patient identifier, prefixed with `pat_`.
</ResponseField>

<ResponseField name="mrn" type="string">
  The patient's Medical Record Number (MRN), auto-generated by the platform at
  record creation.
</ResponseField>

<ResponseField name="first_name" type="string">
  Patient's legal first name.
</ResponseField>

<ResponseField name="last_name" type="string">
  Patient's legal last name.
</ResponseField>

<ResponseField name="date_of_birth" type="string">
  Patient's date of birth in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="gender" type="string">
  Administrative gender: `male`, `female`, `other`, or `unknown`.
</ResponseField>

<ResponseField name="email" type="string">
  Patient's email address. Present only if one was provided at creation or via
  a subsequent update.
</ResponseField>

<ResponseField name="phone" type="string">
  Patient's phone number. Present only if one was provided at creation or via
  a subsequent update.
</ResponseField>

<ResponseField name="address" type="object">
  Patient's primary mailing address. Contains `line1`, `line2` (optional),
  `city`, `state`, and `zip`. Present only if an address was supplied.
</ResponseField>

<ResponseField name="external_id" type="string">
  Your internal system's identifier for this patient, if one was provided.
</ResponseField>

<ResponseField name="status" type="string">
  Current record status: `active`, `inactive`, or `deceased`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp indicating when this patient record was originally created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent modification to this record.
</ResponseField>

### Error Responses

| Status | Error Code     | Description                                                              |
| ------ | -------------- | ------------------------------------------------------------------------ |
| `404`  | `not_found`    | No patient exists with the given ID.                                     |
| `403`  | `forbidden`    | Your API key does not carry a sufficient scope to read patient records.  |
| `401`  | `unauthorized` | Missing or invalid `Authorization` header.                               |

#### 404 Not Found

```json
{
  "error": "not_found",
  "message": "Patient not found",
  "status": 404,
  "request_id": "req_abc"
}
```

<ResponseField name="error" type="string">
  Machine-readable error code. Use this value for programmatic error handling.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable description of the error.
</ResponseField>

<ResponseField name="status" type="integer">
  HTTP status code echoed in the response body for convenience.
</ResponseField>

<ResponseField name="request_id" type="string">
  Unique identifier for this API request. Include this value when contacting
  support to speed up diagnosis.
</ResponseField>

<Tip>
  If you're unsure whether a patient exists, use
  `GET /patients/search?external_id=YOUR_ID` or
  `GET /patients/search?mrn=MRN-XXXXX` before attempting a direct lookup.
  This avoids hard-coding assumptions about ID values in your integration logic.
</Tip>

<Warning>
  Patient IDs are immutable and never reused. If you receive a `404` for an ID
  that previously returned data, the record may have been merged into another
  patient — check the `/patients/search` endpoint using the `mrn` or
  `external_id` to locate the surviving record.
</Warning>
