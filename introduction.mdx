CodeCrafting Hub gives developers a unified REST API to build modern healthcare applications — from patient intake and clinical documentation to claims submission and payment reconciliation. Whether you're embedding EHR workflows into an existing product, building a standalone billing engine, or integrating with payers and scheduling systems, CodeCrafting Hub provides the compliant, production-ready infrastructure you need so you can focus on your application logic instead of healthcare plumbing.

## What You Can Build

CodeCrafting Hub exposes six core capability areas through a single, versioned REST API rooted at `https://api.codecraftinghub.com/v1`:

- **Patient Management** — Create, update, search, and merge patient records with full demographic and insurance detail.
- **Clinical Documentation** — Capture structured encounters, SOAP notes, diagnoses (ICD-10-CM), and procedures (CPT/HCPCS).
- **Claims & Billing** — Generate, validate, and submit professional and institutional claims to payers; track adjudication status in real time.
- **Eligibility Verification** — Run real-time 270/271 eligibility checks against hundreds of payers before every encounter.
- **Scheduling** — Manage appointments, provider availability, and patient reminders across facilities and care teams.
- **Payments & ERA** — Reconcile electronic remittance advice (ERA), post payments, and manage patient balance collection.

## Who It's For

CodeCrafting Hub is designed for engineering teams building on top of healthcare data:

- **EHR integration developers** connecting third-party clinical tools to existing health record systems.
- **Practice management ISVs** that need scheduling, billing, and reporting in a single API contract.
- **Revenue cycle management (RCM) vendors** automating claims workflows from charge capture through payment posting.
- **Digital health startups** that want HIPAA-compliant patient data management without standing up their own infrastructure.

## Compliance & Standards

<Note>
  CodeCrafting Hub is fully **HIPAA-compliant**. All data is encrypted in transit (TLS 1.2+) and at rest (AES-256). Your Business Associate Agreement (BAA) is available and required before accessing production patient data.
</Note>

Every resource in the API maps to **FHIR R4** semantics. You can interact with native CodeCrafting Hub JSON payloads or request FHIR-formatted responses by setting the `Accept: application/fhir+json` header on any request. This means your integration stays portable and interoperable with any other FHIR-capable system.

<Tip>
  If you're new to FHIR R4, the [HL7 FHIR R4 specification](https://hl7.org/fhir/R4/) is the authoritative reference. CodeCrafting Hub profiles and extensions are documented in the API reference.
</Tip>

## Authentication Overview

All API requests authenticate with a **Bearer token** passed in the `Authorization` header:

```http
Authorization: Bearer YOUR_API_KEY
```

You manage API keys in the [CodeCrafting Hub Dashboard](https://dashboard.codecraftinghub.com). Sandbox keys (prefixed `cch_test_`) only reach sandbox infrastructure; production keys (prefixed `cch_live_`) route to production. Never embed production keys in client-side code or public repositories.
